# Bank-Loan-Dashboard
## Recommended Structure and Order
### 1.	Project Title / Headline

Bank Loan Portfolio Performance Dashboard
A data-driven view of loan distribution, revenue generation, and bad debt analysis & built to help leaders monitor performance, identify patterns, and enhance loan quality.

### 2.	Short Description / Purpose
 
This dashboard reveals how people access, use, and repay loans, turning financial data into insight.
Its purpose is to help decision-makers understand customer behavior, improve lending quality, and support better financial outcomes.

### 3.	Tech Stack
List the key technologies used to build the dashboard.

The dashboard was built using the following tools and technologies:

• Micosoft Excel – Primary tool for dataset preparation, analysis, and dashboard reporting.

• Pivot Table/Pivot Charts – Used to summarise loan metrics, segment data, and generate dynamic visuals

• Advanced Formulas – (IF, VLOOKUP/XLOOKUP, SUMIFS, COUNTIFS, INDEX–MATCH, TEXT Functions) – Applied to calculate KPIs, classifications,     ratios, and performance flags.

• Conditional Formatting – Highlighted trends, risk areas, good vs bad loan behaviours, and performance alerts

• Data Validation & Dropdown Controls – Enabled interactive filtering and user-friendly selections

• Excel Charts (Bar, Pie, Line, Gauge-like indicators) – Converted data into visual insights

• Slicers & Timelines – Provided interactive navigation for time periods, loan types, and customer segments

• File Format – .pbix for development and .png for dashboard previews.

### 5.	Features / Highlights
The best dashboard explanation format. 
•	Business problem
•	Goal of the dashboard
•	Walk through of key visuals (briefly!)
•	Business impact & Insights

•	Business Problem
Banks handle growing loan volumes but struggle to track approval trends, identify risky applicants, and monitor repayment behaviour effectively. Managers often rely on fragmented reports, making it hard to spot default risks, performance gaps, and lending opportunities in time.

•	Goal of the Dashboard

To provide a single, interactive view of loan performance that helps decision-makers:

1. Monitor approvals vs rejections
2. Identify risky customers
3. Analyse loan distribution patterns
4. Track repayment performance and defaults

Ultimately, the goal is to support smarter credit decisions and improve portfolio health.


•	Walkthrough of Key Visuals

1. Overall KPIs Cards: Show quick metrics like total loans, approval ratio, repayment rate, and default percentage.
2. Loan Status Chart: Compares approved vs rejected cases for quick performance understanding.
3. Income vs Loan Analysis: Visualises how applicant income relates to approval patterns.
4. Credit Score/History Bar Chart: Highlights how credit history influences outcomes.
5. Segment Slicers (Gender, Education, Property Area): Enable filtering to analyse behaviour by customer types.
6. Trend/Comparison Visuals: Display year/month-wise loan patterns to understand shifts.

These visuals turn raw data into easy-to-read insights supporting fast analysis.


•	Business Impact & Insights

1. Marketing Optimization: Helped target the right customer profiles for better loan conversions.
2. Strategic Expansion: Highlighted high-potential regions and customer segments for growth.
3. Risk Assessment: Identified risky borrower patterns to improve screening and recovery.
4. Regional Analysis: Revealed location-based performance gaps for smarter resource planning.
